About the MATLAB app

The MATLAB application is a function that processes data by operating with a GUI instead of typing MATLAB commands .
Click "Apps" in the toolstrip.
There are as many items as the number of items that the app can display.
You can see all by clicking the lower triangle to the right.
Applications related to the contents of this tool exercise include "classification learning device", "regression learning device", and "deep network designer".
In this text, we will take a look at "Classification Learner" and try it out.

Load the dataset

Like last time, I would like to use the iris data set.
Please refer to the previous lecture material " text12_iris.html " for details of this dataset .
To load the data, I just had to run a command like this:
load fisheriris;

Build a discriminator using the MATLAB app "Classification Learner"

Start a session

Find and click "Classification Learner" from "Apps" on the Toolstrip.
If you use MATLAB Online , the MATLAB app is only available in Chrome .
If you use another browser, please switch to Chrome .
If you can't use Chrome in your home environment, you can use the remote desktop service provided by the university.
When the Classification Learner app is launched, click "New Session".
If the workspace is in a clean state with only meas and species , I think that meas is automatically selected as the analysis target variable (dataset variable) .
If meas is not selected for the dataset variable, select meas .
Next, look at the "Response" panel. By default, "From dataset variable" is selected, so select "From workspace variable".
"From data set variable" is used when the feature quantity and class information are combined into one variable.
Since it is divided into meas and species this time , select "From workspace variable".
If you set "Response" to "From workspace variable", I think species will be selected automatically . If that doesn't happen , select the species yourself.
In the "Validation" panel on the right side, you can decide the ratio of what is used as training data and how much is used as test data.
For now, let's leave this as it is and click "Start Session".
You can now load the data into your MATLAB app.
(After starting the session, if you want to build a discriminator for different data, click "New Session" in the app's toolstrip.)

Observe the data

When the session starts, you will see a scatter plot of the data.
It is a diagram that visualizes the combination of features that I did last week.
You can change the combination of features by selecting "Predictor" on the right.
Next, select "Plot" in the tool strip of the application and click "Parallel coordinate plot".
Then you can observe the data using the parallel plots you did last week.

Build discriminator on support vector machine

Let's take a look at the "model type" in the app's toolstrip.
Classification learners are listed here.
Not all are shown due to the large number. Please click the lower triangle to display all and see the contents.
You can see that the methods such as (decision) tree, discriminant analysis, support vector machine are lined up.

Let's do one. Select "Linear SVM " and click.
"Linear SVM " constructs a classifier with a support vector machine without mapping it into a higher dimensional space.
It is called a linear SVM because this does not project data to higher dimensional space with kernels (i.e. using "1" as a kernel, thus it is called a "linear" kernel).
The number of linear discriminants has increased in the "History" section. Do you see the word "draft" on the right?
This means that the corresponding discriminator has not been evaluated.
Click once on the new SVM and then click the Learn button (green triangle) in the toolstrip of the app window.
If your MATLAB license supports parallel computing and you have enabled "use parallel" in the toolstrip, the preparation of the parallel pool needed for parallel computing begins.
This preparation will take some time.
Once the parallel pool is prepared, it can be used next time without preparation.
However, it will be released if you do not use it for a while, and you will need to prepare it before you can use it again.
After the calculation, the classifier is built and the correct answer rate is also displayed. It was 96.7 %.
In addition, standardization of data is performed by default.
For more information, check "Advanced settings" in the app's toolstrip.
It will be helpful as you hover your mouse cursor over the name of the item to get the tip help.

Evaluate the discriminator

There is a button on the toolstrip of the app called "confusion matrix" .
Let's press this. Then, the 3 by 3 matrix will be colored and displayed.
The horizontal and vertical axes are also carefully written. The horizontal axis is the "predicted class" and the vertical axis is the "true class."
As you can see from the name of this axis, this matrix is used to make predictions and actual comparisons for each classification of the data.
The number is the number of data corresponding to the combination of "predicted class" and "true class".
The total number of data is the total number of data. The data this time was 150.
Will it be 150 if added properly ?
This matrix is diagonal when all the data have been classified correctly.
The confusion matrix this time is almost diagonal. So it seems that they can be classified almost correctly.
As for the tendency of mistakes, it is easy to confuse versicolor with virginica .

Build SVM using PCA as preprocessing

Now, let's construct the discriminator using Support Vector Machine after dimensionally compressing the data using the principal component analysis that we did in the last week's exercise.
There is a button called PCA in the tool strip of the application, so click it.
Select the Enable Principal Component Analysis check box to enable preprocessing by principal component analysis.
You can choose between indirect designation by the cumulative contribution ratio and direct designation by the number of components.
Let's try it company-wide here. Let's try with "Specify explanation variance", 95%.
"Characteristic 2/4" is written in the history column. It shows that the four features are compressed in two dimensions.
Please review how many 95% of the cumulative contribution rate calculation results from the previous exercise apply.
After learning, it was 96.7 %.

Construct classifiers other than linear SVM

You can choose various discriminators, so let's try various ones.
First, let's disable PCA .
From the model types, select and click All Quick Learning, then click Learn from the app's toolstrip.
"All quick learning" is a button to try various techniques that can build a classifier in a relatively short time.
After clicking the "Learning" button, it will be expanded to the name of the concrete discriminator and learning will begin.
If parallel calculation is enabled, these will be processed in parallel.
The linear SVM was the bes after all. The number of correct answers is emphasized by a white square is the proof that it is the first place.
Please try to find "all learning" in the model types.
This is a button that applies a set of algorithms, except for hyperparameter tuning.
Let's click.
...Hmm? In this case, linear discrimination seems to be the best with 98% accuracy.
Linear discriminant is a well-known method of constructing a linear classifier considering the average of data of each class and the spread of data.
Also called Fisher's (linear) discriminant analysis.
If you are curious, check the help with the following command.
% doc 判別分析分類
doc Discriminant Analysis Classification
On the other hand, SVM is a method of looking only at the boundaries of data.
On the contrary, the algorithms that are compatible with each other may give a glimpse of the nature of the data.

Export discriminator

The classifier built up to this point is currently valid only within the application.
To use it in another part of MATLAB , do an "export".
In this case, I am concerned about the best linear discrimination , but let's export the linear SVM treated in the theory-based course .
Click the linear SVM in the history column .
Next, there is "Export" in the tool strip of the application, so click the lower black triangle.
" Export plot to figure " is used to output the figure created on the app.
"Generate function" outputs with the procedure MATLAB code for generating the target discriminator . Shall we try it?
What about the generated code? This is a traditional MATLAB program in the form of m- files .
It looks different, but the program is the same. If you run it, it will work. I will change it a little if necessary.
"Export Model" exports the discriminator as an object to the workspace as is.
There are two more options depending on whether or not to retain the data used for learning. Here, select "Export Model" (the one that is not compact) and click it.
The name can be trainedModel .
Return to the main screen of MATLAB and check the workspace, trainedModels was added.

Summary

In this text, I tried to use the Classification Learner app for classification problems.
If you don't want to classify but want to estimate values, use Regression Learner.
As you probably noticed, almost no code is written.
I just wrote a little bit to read the iris data and use the discriminator I made on the command line.
It's so convenient, isn't it?
(Moreover, it was possible to read the data only by operating the mouse when importing from xls(x) or csv .)
There may be occasions when you want to try machine learning a bit as you proceed with your research.
In that case, the MATLAB app is very convenient.
As I've seen in this text, you can use it for the time being without having to learn the language from scratch or how to use the library.
It's a good idea to use it while being careful not to let it be used as a tool.
